Ex01: Template Engine

Django's template engine is a powerful system that allows developers to create HTML templates with dynamic content. It provides a way to render HTML while integrating Python-like logic for more flexibility.

Blocks

Blocks are used to define sections of a template that can be overridden in child templates. For example:

{% verbatim %}{% block content %}{% endblock %}{% endverbatim %}

This enables you to create a base template with common elements, while allowing individual pages to define their own unique content.

Loops (for ... in)

Loops in Django templates allow you to iterate over a list or queryset. Here's an example:

{% verbatim %}{% for item in item_list %}
    
{{ item }}

{% endfor %}{% endverbatim %}

Control Structures (if statements)

Django templates allow conditional statements, like if statements, to control the flow of content based on certain conditions:

{% verbatim %}{% if user.is_authenticated %}
    
Welcome, {{ user.username }}!

{% else %}
    
Please log in.

{% endif %}{% endverbatim %}

Context Variables

Context variables are dynamic values passed from the view to the template. They are accessible using double curly braces {% verbatim %}{{ variable_name }}{% endverbatim %} and are used to inject data into the template:

{% verbatim %}{{ user.username }}{% endverbatim %}

For example, user.username could display the current username if it's passed in the context of the view.

Summary

With the Django template engine, you can render HTML dynamically, making it an essential tool for creating flexible web applications. Using blocks, loops, conditional statements, and context variables helps keep templates clean, reusable, and efficient.
