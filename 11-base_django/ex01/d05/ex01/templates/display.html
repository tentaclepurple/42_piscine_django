Ex01: Display Process of a Static Page

When a user requests a static page in Django, several steps are involved in processing and displaying that page. Here’s an overview of how Django handles this process, from receiving a request to sending back an HTML response:

1. URL Routing

When a user enters a URL (e.g., /ex01/display) into their browser, Django’s URL dispatcher matches this URL to a specific view function in urls.py. This view function is responsible for processing the request and rendering the appropriate response.

2. View Function

Once the URL matches a view, Django calls the view function. This function is usually defined in views.py. The view can either return a pre-defined HTTP response or render a template using the render function. For example:

def display_process_page(request):
    return render(request, 'display.html')

3. Template Rendering

The view function calls Django’s template engine to render the HTML template associated with the view (in this case, display.html). The template is processed, filling in any dynamic content or variables as needed, and generates the final HTML output.

4. HTTP Response

Once the template is rendered into HTML, the view function wraps it in an HTTP response object and sends it back to the client. The client (usually a browser) receives the response and displays the HTML page.

Summary

The entire process is designed to be efficient and flexible. Django’s use of URL routing, view functions, and templates makes it easy to create both static and dynamic pages.
